# Архітектура LiveKit Meet

Проект повинен використовувати **Bun** як менеджер пакетів і мати структуру **Bun монорепозиторію**.

Частина **Next.js** знаходиться в директорії **`site/`**.

## Структура монорепо

- **`site/`** — додаток Next.js (сторінки, API, стилі, публічні файли).
- **`book/`** — документація (mdbook).
- Корінь репозиторію — конфіги (Playwright, Prettier, GitHub Actions), скрипти запускають команди в `site/` через `bun run --cwd site ...`.

## Bun

- **Встановлення залежностей:** `bun install`
- **Додати пакет:** `bun add <пакет>` або `bun add -d <пакет>` для dev
- **Запуск скриптів:** `bun run <script>` або скорочено `bun <script>` (наприклад `bun dev`, `bun build`)
- Lock-файл: `bun.lockb` (комітити в репозиторій)
