# Задача

В директорії book
повинна бути описана для користувачів функціональність системи у вигляді книги
Функціональність продукту повинна бути описана україньскою мовою
Назви директорій з розділами, повинні бути кирилецею

## Структура книги

і в ній всередині буде MD файл зі смістом, а директорії будуть розділи
а в розділах буде опис функціональності

тобто ми функціональність системи опишемо у вигляді книги

## Тести функціональності

В кожному розділі книги (відповідній директорії), створена директорія test, в якій знаходяться Playwright тести для фронтенд функціональних можливостей

та BUN Test runner тести для не візуальних блоків коду

шлях до тестів буде наприклад, такий

book/01-вступ/test/перевірка-входу.spec.ts

самі тести повинні включатись до сторінок опису з функціональними можливостями,
потрібно щоб вставка була з спеціальними коментарами маркерами
наприклад {{#include ../test/specs/auth.spec.ts:login_test}}

Секції з тестами повинні бути обгорнуті в HTML-елемент <details> без атрибута open, щоб за замовчуванням були схлопнуті. Користувач бачить лише рядок «Тести функціональності» (з іконкою трикутника); по кліку блок розгортається і показує код тестів.

## Архітектура

Точка входу (зміст книги) це файл SUMMARY.md

Для візуалізації цієї книги повинен бути створений book.toml

```toml
# MdBook — конфігурація книги
# Документація: https://rust-lang.github.io/mdBook/

[book]
title = "Книга функціональності"
language = "uk"
authors = []

# Джерела тут же (поточна директорія)
src = "."

# Результат збірки — статичні HTML у піддиректорії
[build]
build-dir = "html"
create-missing = false

[output.html]
default-theme = "light"
preferred-dark-theme = "navy"
smart-punctuation = true
mathjax-support = false
no-section-label = false
search = { enable = true }
```

для користувача книга буде відображатись за допомогою mdbook:

```bash
cd book && mdbook serve
```
